# 📜 Квиз по истории России

Интерактивный тренажёр для подготовки к экзамену. **234 вопроса** в 22 темах:
Древняя Русь → Российская империя → СССР. Неверный ответ — тема раскрывается.

🔗 **Демо:** https://lisenokf.github.io/historyRussianquiz/

Работает офлайн, без сборки и зависимостей. Чистая статика (HTML + CSS + JS), можно
установить на телефон как приложение (PWA). Есть личная статистика и общий рейтинг.

## Возможности
- ❓ Квиз по темам или «всё подряд», с разбором каждой ошибки.
- 👤 **Запоминание пользователя** — имя вводится один раз и хранится в cookie + localStorage
  (никакого сервера, узнаём тебя на этом устройстве автоматически).
- 📊 **Личная статистика** — пройдено квизов, средний и лучший результат, результаты по темам.
  Хранится локально в браузере.
- 🏆 **Общий рейтинг (лидерборд)** — результаты всех игроков в одной таблице.
  Требует разовой настройки бесплатного Google Apps Script (см. ниже).

## Структура проекта
```

├── manifest.webmanifest     # иконка/название при установке как PWA
├── htaccess                 # конфиг Apache (для хостингов вроде reg.ru; на GitHub Pages не нужен)
├── assets/
│   ├── css/
│   │   └── styles.css       # все стили
│   └── js/
│       ├── questions.js     # БАЗА ВОПРОСОВ — её и редактируй
│       ├── identity.js      # «авторизация по куки»: имя + стабильный id
│       ├── stats.js         # личная статистика (localStorage)
│       ├── leaderboard.js   # подключение к общему рейтингу
│       └── quiz.js          # логика квиза и экранов
└── apps-script/
    └── Code.gs              # код мини-API для рейтинга (Google Apps Script)
```

## Хостинг на GitHub Pages
Сайт уже опубликован. Чтобы включить/проверить Pages в своём форке:
1. **Settings → Pages**.
2. **Source:** Deploy from a branch.
3. **Branch:** `main` (или `develop`), папка `/ (root)` → **Save**.
4. Через минуту сайт будет на `https://<логин>.github.io/<репозиторий>/`.

Все пути в проекте относительные, поэтому проект работает и в подпапке
(`/historyRussianquiz/`), и на собственном домене — менять ничего не нужно.

## Лидерборд (общий рейтинг)
GitHub Pages — это только статика, без базы данных, поэтому общий рейтинг хранится
во внешнем бесплатном хранилище — **Google Таблице**, к которой подключён скрипт-API.

Пока `LEADERBOARD_API` пуст, приложение работает нормально, просто на вкладке
«Рейтинг» показывает «не настроено». Чтобы включить рейтинг:

1. Создай новую **Google Таблицу** (sheets.new).
2. **Расширения → Apps Script**. Удали заготовку и вставь содержимое
   [`apps-script/Code.gs`](apps-script/Code.gs). Сохрани.
3. **Развернуть → Новое развёртывание**:
   - Тип: **Веб-приложение**;
   - «Выполнять от имени»: **Я**;
   - «У кого есть доступ»: **Все** (Anyone).
4. Подтверди доступ Google, скопируй **URL веб-приложения** (заканчивается на `/exec`).
5. Открой [`assets/js/leaderboard.js`](assets/js/leaderboard.js) и вставь URL:
   ```js
   const LEADERBOARD_API = 'https://script.google.com/macros/s/AKfyc.../exec';
   ```
6. Закоммить и запушь — рейтинг заработает. Результаты накапливаются в листе
   `leaderboard` таблицы; в рейтинге показывается лучший результат каждого игрока.

> Технические детали: запись идёт `POST`-ом с `Content-Type: text/plain` (это «простой»
> запрос без CORS-preflight, который Apps Script корректно принимает), чтение — `GET ?action=top`.

## Редактирование вопросов
Открой [`assets/js/questions.js`](assets/js/questions.js) — формат очевиден:
```js
{
  q: "Текст вопроса?",
  options: ["вариант 1", "вариант 2", "вариант 3", "вариант 4"],
  correct: 0,            // индекс правильного ответа (0 = первый)
  explain: "Объяснение, которое раскроется при ответе."
}
```
Варианты ответов и сами вопросы перемешиваются автоматически.

## Локальный запуск
Так как это статика, достаточно отдать файлы любым веб-сервером (открывать через
`file://` нежелательно — не загрузятся внешние `.js`). Например:
```bash
python3 -m http.server 8000
# затем открой http://localhost:8000
```

## Прочее
- **PWA**: `manifest.webmanifest` позволяет «установить» сайт на телефон/рабочий стол.
- **`htaccess`**: конфиг для Apache-хостингов (reg.ru и т.п.) — кодировка, MIME, кэш.
  На GitHub Pages не используется, оставлен для совместимости с другими хостингами.
  При заливке на Apache переименуй файл в `.htaccess` (со скрытой точкой в начале).
